## 方法與限制

- **價格資料截止日**：本次執行時查得 `daily_prices` 最新日期為 **2026-07-16**（資料庫回補進行中，此為讀取當下的快照，之後可能再往後延伸；重跑本報告腳本會拿到更新的截止日）。交易日曆以 `taiex_daily` 的日期序列為準，並截到上述價格截止日。
- **事件定義**：對每檔股票，`trust_net > 0` 連續 N 天（N=3/5/10）首次達標的當日為事件日 T，起始日 S=T-N+1；`trust_net = 0` 或 `< 0` 視為中斷。賣方（連續 N 天 `trust_net < 0`）鏡像處理。同一段較長連買（例如連 15 天）會同時貢獻 N=3、5、10 各一次事件，三組獨立統計，不去重，此為標準做法。
- **剔除規則**：事件必須在 [S-10, T+20] 整個窗口內，該股每一個交易日都有收盤價可算報酬（即該窗口任何一天缺值就整筆事件剔除），並且窗口不可超出資料庫的交易日曆邊界。各組事件剔除率約 17%–21%（詳見下表 raw→eligible）。
- **報酬計算**：個股日報酬（`close` 之 pct_change）**減去**同日 TAIEX 日報酬，得到「市場調整後日超額報酬」；只扣大盤，**未扣除產業或規模因子**，故若某事件恰好發生在同族群類股齊漲齊跌時，超額報酬仍會殘留產業效應。多日累積一律採**日超額報酬直接加總**（CAR，非連乘複利），已在下表註明。
- **事件重疊與統計顯著性**：同一檔股票的連買事件在時間上常前後相鄰或重疊、不同股票的報酬也會同時受同一波大盤/類股輪動影響，橫斷面相關會使樣本「有效自由度」遠低於原始事件數，因此下列 t 值**僅供參考量級，不可直接當成獨立樣本的嚴格顯著性**。
- **稀有事件提醒**：`institutional_flow_daily` 中 `trust_net = 0` 佔全部有效觀測值約 **81.5%**（1,057,835 / 1,298,500）——多數個股多數交易日投信根本不交易。因此「連續 N 天淨買超」本身就是稀有事件，樣本數（見下表）已如實列出，N=10 事件數明顯少於 N=3。
- **因果限制**：事件研究只能提供「時間先後順序」的證據（先發生的變數 → 後發生的變數之間有無超額報酬），相關不等於嚴格因果；仍可能有第三因素（例如共同的基本面消息）同時驅動股價與投信買賣。

## 因果結論

**資料支持的方向：偏向「股價→資金流」，且買賣兩邊機制不對稱，資金流本身對未來股價幾乎沒有正向預測力（賣方訊號反而略有預測力，但方向對投資人不利）。**

1. **買方連買**：事件前報酬幾乎持平（僅弱到中等程度的追價跡象，Q4 佔比 27%–31% 略高於 25% 基準），事件後（T+1~20）轉為穩定的負超額報酬（-1.0% 上下，t 值 -4 至 -9，正報酬比例僅 41%–42%）。也就是說，看到投信連買 N 天才進場，統計上**平均反而是小幅落後大盤**，不是「還來得及」，接近「利多出盡／資金動能耗盡」的反轉型態，而非動能延續。
2. **賣方連賣**：事件前股價已經顯著落後大盤（中位數約 -1.5%），這是「股價先跌、投信才連續賣超」的較清楚證據（雖然反向檢驗的分位集中度並不誇張）；更關鍵的是事件後跌勢**持續**（T+1~20 均值 -1.4%~-1.7%，t 值 -6 至 -13，正報酬比例僅 36%–39%），顯示連賣訊號比連買訊號有更明確（雖然仍偏弱）的「資金流→股價」預測力，方向是繼續走弱。
3. **N=3/5/10 遞增或衰竭**：買方 T+1~20 負報酬幅度在 N=3/5/10 間大致持平（-1.04%／-1.03%／-1.08%），沒有隨連買天數拉長而加碼轉強或轉弱的清楚趨勢，說明「連買越久越看好」的直覺缺乏數據支持。賣方則有較明顯的加深趨勢（-1.42%／-1.37%／-1.69%，N=10 跌幅最深、正報酬比例最低），顯示連賣天數越長，後續補跌的力道傾向越強（動能延續而非衰竭）。
4. **實用含義**：看到「連買 3 天」才進場，統計上不僅不算晚，反而平均會buy在偏高點附近，之後 20 個交易日平均市場調整報酬為負；連買 5、10 天也是同樣結論，天數拉長並未改善勝率。反過來，看到「連賣 N 天」，尤其 N=10，數據顯示後續繼續弱勢的機率較高（正報酬比例僅 36.5%），可作為風險提示訊號，但預測力仍屬統計上偏弱、需搭配其他資訊使用，不構成投資建議。

（以上為市場調整後歷史統計結果，事件高度重疊、t 值僅供量級參考，且相關不等於嚴格因果，僅供研究參考，不構成投資建議。）
